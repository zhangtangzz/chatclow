3. 核心架构：责任链模式

整个对话流程被拆成 7个独立Step，由 ChatChainImpl 按 @Order 顺序驱动。Spring 自动收集所有 ChatChainStep 实现类并排序注入，新增 Step 只需写类加注解。

双向路由机制

ModelCallStep 和 SyncAiCallStep 同为 @Order(30)，通过 shouldSkip() 互斥：

  • 流式请求 → ModelCallStep 执行，SyncAiCallStep 跳过
  • 非流式请求 → SyncAiCallStep 执行，ModelCallStep 跳过

数据传递机制

所有 Step 共享同一个 ChatContext 对象（Java 引用传递）。前一步 ctx.setXxx()，后一步 ctx.getXxx() 就能读到。不需要返回值、不需要全局变量。

6. 功能优化与改进记录

消除FC代码重复

新增 ToolCallHelper，将 FunctionCallingStep（流式）和 SyncAiCallStep（非流式）中 ~80 行重复的消息构建+工具执行逻辑提取为公共方法，两边各自减少 80+ 行代码。

ChatRole 常量

新增 ChatRole 常量类，统一管理 "system"、"user"、"assistant"、"tool" 四种角色字符串，替换 5 个文件中的硬编码。

FunctionExecutorImpl 重构

switch-case 路由 → Map<String, Function> 注册模式；手写 JSON 字符串 → Jackson ObjectMapper 序列化。新增工具只需一行 registry.put()。

ChatContext 清理

删除未使用的 messages 字段和 addMessage() 方法，注释从冗长 HTML 格式简化为紧凑结构化注释。

日志统一 + 线程池

全部 e.printStackTrace() 和 System.err.println() 替换为 SLF4J Logger，涉及 9 个文件；所有 new Thread() 替换为 sseExecutor 线程池，统一管理异步任务并发。

注释全面精简

所有 Chain Step 统一为三步注释格式：
① 第几步 / ② 参数从哪里来 / ③ 调用什么方法。去掉冗长的 HTML 类比、开发过程描述。

清理冗余代码

删除 DemoController、DemoVectorStore、/test-chunk 测试接口、test_sse.html 等试验/测试代码。

代码量变化

19个文件变更 -1644 行 / +578 行，净减少约 1000 行代码。

API 密钥 AES 加密存储

新增 AesUtil 工具类，对 AI 模型的 API 密钥进行 AES-128/ECB/PKCS5Padding 加密。保存时加密入库、展示时掩码（前4字符+****）、使用时自动解密，端到端保护密钥安全。

SSE 错误通知 + 前端终止按钮

对话异常时向前端发送 error SSE 事件（含错误信息），前端通过 AbortController 中断 fetch 请求。新增红色终止按钮，用户可随时手动停止卡死的 AI 回复。

SSE 断线重连机制

连接异常断开后显示重试横幅，保留上次消息内容。用户点击"重新发送"即可一键重试，不需要重新输入，不丢失对话上下文。

级联删除修复

删除知识库时级联清理文档、切片和向量存储数据；删除模型时同步清理引用关系；删除智能体时级联删除关联工具定义。避免数据库和向量存储产生脏数据。

8. 关键技术要点

SPI 可插拔向量存储

通过 ChatClowVectorStore 接口 + VectorStoreFactory，支持 MySQL / MongoDB / pgvector 三种后端。按知识库级路由，配置文件切换，不改一行代码。

SSE 流式输出

用 OkHttp 发起 stream=true 的 HTTP 请求，逐行解析 data: {...} 事件，通过 SseEmitter 逐 token 推送给前端。tool_calls 在流式模式下是分片到达的，通过 StreamToolCall 按 index 拼装。

双路混合检索

向量检索（余弦相似度，语义匹配）+ 关键字检索（MySQL LIKE + 中文分词，精确匹配），结果合并去重后再由专用 Re-rank 模型重新打分排序，兼顾语义理解和精确匹配。

Function Calling

AI 返回 tool_calls → 解析函数名+参数 → FunctionExecutor 执行 → 结果以 role=tool 消息回传 → 二次请求（不带 tools 避免死循环）。流式模式通过 SSE 事件通知前端工具调用状态。

AES 加密存储

API 密钥通过 AES-128/ECB/PKCS5Padding 加密后存入数据库。前端展示时自动掩码（只显示前4位），后台使用时自动解密。旧明文密钥在首次编辑保存时自动转为加密存储。

SSE 错误处理体系

异常时发送 error SSE 事件告知前端具体错误信息；前端 AbortController 终止请求 + 重试横幅一键重发 + ChatServiceImpl 兜底异常捕获，三层保障防止对话卡死无反馈。
