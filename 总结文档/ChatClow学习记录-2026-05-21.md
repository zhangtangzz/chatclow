## 二、安全&稳定性 三件套

### 2.1 JWT 登录认证 ✅

> 核心思路：用户登录成功后签发 Token，后续请求通过 Header 携带 Token 进行身份验证。

#### 涉及文件：

| 文件 | 作用 |
|------|------|
| `util/JwtUtil.java` | JWT 工具类，负责生成/解析/验证 Token |
| `controller/LoginController.java` | 登录接口 `POST /api/auth/login` |
| `interceptor/JwtInterceptor.java` | 拦截器，验证每个请求的 Token |
| `config/WebMvcConfig.java` | 注册拦截器 + 跨域配置 |

#### 认证流程：

```
前端传 username + password
    → 查库验证
        → JwtUtil.generateToken() 签发Token
            → 前端存储Token

后续请求: Header: Authorization: Bearer <token>
    → JwtInterceptor 验证
        → 放行 / 返回401
```

#### 关键代码 —— Token 生成：

```java
public static String generateToken(Long userId, String username) {
    return Jwts.builder()
            .claim("userId", userId)
            .claim("username", username)
            .setIssuedAt(new Date())
            .setExpiration(new Date(System.currentTimeMillis() + EXPIRE_TIME))
            .signWith(SignatureAlgorithm.HS256, SECRET_KEY)
            .compact();
}
```


### 2.2 全局异常处理 ✅

> 使用 `@RestControllerAdvice` + `@ExceptionHandler` 统一捕获异常，返回友好提示而非堆栈信息。

#### 异常处理清单：

| 异常类型 | 触发场景 | 返回格式 |
|---------|---------|----------|
| `MethodArgumentNotValidException` | @Valid 校验失败时 | `{code:400, msg:"具体错误"}` |
| `BindException` | 参数绑定错误 | `{code:400, msg:"参数绑定失败"}` |
| `NullPointerException` | 空指针异常 | `{code:500, msg:"系统繁忙，请稍后重试"}` |
| `IllegalArgumentException` | 非法参数异常 | `{code:400, msg:"参数不合法"}` |
| `Exception (兜底)` | 其他所有未捕获异常 | `{code:500, msg:"服务器内部错误"}` |

### 2.3 参数校验 ✅

> 基于 JSR-303 规范（Hibernate Validator 实现），通过注解约束实体字段。

#### 校验注解一览：

| 注解 | 作用 | User.java 使用场景 |
|------|------|-------------------|
| `@NotBlank` | 不能为 null / 空串 / 纯空格 | username, email, password |
| `@Email` | 必须是合法邮箱格式 | email 字段 |
| `@Size(min, max)` | 字符串长度限制 | username(3~20), password(6~32) |
| `@NotNull` | 不能为 null | role 角色 |

#### ⚠️ 关键点：

- Controller 接口必须用 **`@RequestBody @Valid`** 才能触发校验！
- 校验失败抛出 `MethodArgumentNotValidException`，被 `GlobalExceptionHandler` 兜住
- Spring Boot 2.x 用 **`javax.validation`**（不是 jakarta）

#### UserController 注册接口改造：

```java
// 改造前：一个个接参数
@PostMapping("/register")
public R<Void> register(@RequestParam String username,
                         @RequestParam String password,
                         @RequestParam String email) { ... }

// 改造后：整体接对象 + 校验
@PostMapping("/register")
public R<Void> register(@RequestBody @Valid User user) {
    boolean success = userService.register(
        user.getUsername(), user.getPassword(), user.getEmail()
    );
    if (success) {
        return R.ok("注册成功!", null);  // 注意双参数写法！
    }
    return R.error("用户名已存在！");
}
```

### 核心代码片段 —— LambdaQueryWrapper 查询：

```java
@Override
public List<AiModel> listEnabled() {
    return aiModelMapper.selectList(
        new LambdaQueryWrapper<AiModel>()
            .eq(AiModel::getStatus, 1)              // where status = 1
            .orderByDesc(AiModel::getCreatedDt)       // order by created_dt desc
    );
}
```

### 核心代码片段 —— 状态切换逻辑：

```java
@Override
public Integer toggleStatus(Long id) {
    AiModel model = aiModelMapper.selectById(id);
    if (model == null) {
        throw new RuntimeException("模型不存在");
    }
    int newStatus = model.getStatus() == 1 ? 0 : 1;  // 1→0 或 0→1
    model.setStatus(newStatus);
    aiModelMapper.updateById(model);
    return newStatus;
}
```

## 四、踩坑记录 & 解决方案

### ❌ 坑 1：Spring Boot 2.6 拦截器白名单失效

- **现象**：登录接口 `/api/auth/login` 被拦截器拦住，返回 401 "未登录"
- **原因**：Spring Boot 2.6 默认路径匹配策略改为 PathPatternParser，导致 `excludePathPatterns` 不匹配
- **解决**：`application.yml` 中添加：
  ```yaml
  spring:
    mvc:
      pathmatch:
        matching-strategy: ant_path_matcher
  ```

### ❌ 坑 2：Java 17 JAXB 缺失

- **现象**：启动报错 `ClassNotFoundException: javax.xml.bind.DatatypeConverter`
- **原因**：jjwt 0.9.1 使用了 `javax.xml.bind`，该包从 Java 9 开始被移除
- **解决**：pom.xml 添加 `jaxb-api`(2.3.1) + `jaxb-runtime`(2.3.1)

### ❌ 坑 3：Bearer 大小写问题

- **现象**：携带 Token 访问仍返回 401
- **原因**：Header 中写了小写 `bearer token`，必须是大写 `Bearer` 开头
- **解决**：`Authorization: Bearer <token>` （B 大写）

### ❌ 坑 4：Token 放错了位置

- **现象**：ApiPost 中带了 Token 还是 401
- **原因**：Token 写在了 Body 标签页，应该放在 Header 标签页
- **解决**：ApiPost → Headers 标签页 → 添加 `Authorization: Bearer <token>`

### ❌ 坑 5：R.ok() 单参数类型冲突

- **现象**：`R.ok("注册成功!")` 返回类型 `R<Void>` 但编译报错
- **原因**：`R.ok(T data)` 单参数版本把参数当 data 处理，String ≠ Void
- **解决**：改用双参数版本 `R.ok("注册成功!", null)`，第一个参数才是 msg

### ❌ 坑 6：Mapper 文件名写错

- **现象**：新建 Mapper 后 IDE 报错
- **原因**：文件名写成 `AiModel.java`（和实体类重名），应为 `AiModelMapper.java`
- **解决**：重命名为 `AiModelMapper.java`

## 五、今日核心知识点

### ▶ JWT (JSON Web Token)

无状态的认证方式，包含三部分：Header.Payload.Signature。服务端用密钥签名验证，无需在服务端存储会话。jjwt 库用于 Java 中的 JWT 操作。

### ▶ HandlerInterceptor

Spring MVC 拦截器接口，`preHandle` 方法在 Controller 执行前拦截请求。适合做登录鉴权、日志记录等横切关注点。

### ▶ WebMvcConfigurer

Spring MVC 配置接口，用于注册拦截器、配置跨域等。配合 `@Configuration` 使用。

### ▶ @RestControllerAdvice

全局控制器增强，配合 `@ExceptionHandler` 实现统一异常处理。是 RESTful API 的标准做法。

### ▶ JSR-303 Bean Validation

Java 标准的参数校验规范。常用注解：`@NotBlank`、`@NotNull`、`@Size`、`@Email`、`@Pattern`。需搭配 `@Valid` 使用。

### ▶ LambdaQueryWrapper

MyBatis-Plus 的条件构造器，用方法引用（`实体类::getField`）构建查询条件，避免手写字段名字符串出错。

### ▶ @RequestBody vs @RequestParam

- `@RequestBody` 接收 JSON 对象（需配合 `@Valid`）
- `@RequestParam` 接收单个查询参数或表单字段
- 两者不能混用在同一场景

### ▶ Spring Boot 版本差异

| | Spring Boot 2.x | Spring Boot 3.x |
|--|---|---|
| validation 包 | `javax.validation` | `jakarta.validation` |
| 路径匹配策略 | 需手动配 `ant_path_matcher` | 默认即可 |
