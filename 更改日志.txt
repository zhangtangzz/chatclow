2026-06-06 SSE 流式输出修复（多轮迭代，最终方案）
-----------------------------------------------

【问题】前端 AI 回复不是逐字流式出现，全文一次性显示
【排查过程】
  1. 后端 Tomcat response.setBufferSize(0) → F12 确认 SSE 已流式 ✅
  2. fetch ReadableStream + async 回调 + setTimeout → 仍不行 ❌
  3. setInterval 缓冲队列 → 引入响应式 bug，逻辑错乱 ❌
  4. await + setTimeout(30ms) → 仍不行 ❌
【根本原因】Vue 3 响应式：原始对象 aiMsg.content += text 绕过 Proxy，Vue 检测不到变化不更新 DOM
【最终方案：requestAnimationFrame 逐字渲染】
  - ChatController.java → sendStream() 加 HttpServletResponse，response.setBufferSize(0)
  - api/chat.js         → 回调同步执行（无 await），速度快
  - views/Chat.vue      → 新增 pendingChars 缓冲 + RAF 渲染循环
    - SSE content 事件 → pendingChars += event.data（快速收集）
    - renderNextChar()  → 每帧取 2 个字符写入 messages.value[idx].content（响应式代理）
    - done/error        → stopCharRender() 清空缓冲 + 取消 RAF
    架构：SSE 只管收字，RAF 独立驱动渲染，两者完全解耦

2026-06-06 03:45:00 SSE 流式前端重写（第6次，独立 ref + setInterval 方案）
-----------------------------------------------------------------------
【改动方案】
- 根本原因：之前 RAF 方案仍可能因 Vue 响应式代理问题导致 msg.content += text 不触发 DOM 更新
- 新方案：用独立的 ref（streamText）驱动渲染，与消息对象完全解耦
  1. SSE content 事件 → rawBuffer（非响应式字符串，最快速度收集）
  2. setInterval 每 30ms → 从 rawBuffer 取 3 个字符写入 streamText.value（触发响应式）
  3. 模板 displayContent(msg, idx) → msg.content + streamText.value（响应式绑定）
  4. done/error → commitStreamText() 将 streamText 提交到 msg.content 并重置
【前端改动】
- views/Chat.vue → 变量：streamText ref + rawBuffer + streamTimer 替代 pendingChars + RAF
- views/Chat.vue → 函数：displayContent / startStreamRender / stopStreamRender / commitStreamText
- views/Chat.vue → 模板：v-html 用 displayContent(msg, idx)，打字指示器 v-if="streaming && !streamText"
- views/Chat.vue → sendMessage：content 事件写 rawBuffer，done/error 调 commitStreamText
